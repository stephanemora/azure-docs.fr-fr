# <a name="power-bi-app-offer"></a>Offre d’application Power BI

|              |                                |
|--------------|--------------------------------|
| Cet article explique comment publier une application Power BI sur la [Place de marché Microsoft AppSource](https://appsource.microsoft.com/).  Une application Power BI regroupe le contenu Power BI personnalisable, notamment les jeux de données, les rapports et les tableaux de bord. Vous pouvez ensuite déployer l’application sur d’autres locataires Power BI via AppSource, effectuer les ajustements et les personnalisations autorisés par le développeur et la connecter à vos propres données. | ![Icône Power BI](./media/powerbi-icon.png) |


Cet article est divisé en trois parties principales :

-   [Conditions préalables](./cpp-prerequisites.md). Exigences techniques et commerciales pour la création et la publication d’une offre d’application Power BI.
-   [Créer une offre d’application Power BI](./cpp-create-offer.md). Comment créer une entrée d’offre d’application Power BI à l’aide du [Portail Microsoft Cloud Partner](https://cloudpartner.azure.com).
-   [Publier une offre d’application Power BI](./cpp-publish-offer.md). Comment soumettre une nouvelle offre sur AppSource en vue de sa publication, et comment mettre à jour une offre existante.


## <a name="publishing-steps"></a>Étapes de publication

Voici les étapes principales pour la publication d’une offre d’application Power BI :

![Étapes de publication d’offre d’application Power BI](media/publishing-steps.png)

Voici le processus de publication d’offre d’application Power BI :

1. Créez une application modèle dans Power BI. Cette action génère une URL d’installation de package, qui représente la ressource technique principale de l’offre. À ce stade, vous devez également promouvoir le package de test en pré-production. Pour plus d’informations, consultez [Que sont les applications modèle Power BI ?](https://docs.microsoft.com/power-bi/service-template-apps-overview). 
2. Collectez ou créez les supports marketing de l’offre, notamment le nom officiel, la description, les logos, etc. 
3. Collectez ou créez les documents juridiques et de support de l’offre : *conditions d’utilisation*, *politique de confidentialité*, *politique de support*, aide de l’utilisateur, etc.
4. Créez l’offre : utilisez le Portail Cloud Partner pour configurer les détails de l’offre, y compris la description de l’offre, les supports marketing, les mentions juridiques, les informations de support et les spécifications des ressources.  Lorsque l’offre est complètement spécifiée, soumettez-la pour publication.
5. Surveillez le processus de publication dans le Portail Cloud Partner.  Pendant cette étape, l’équipe d’intégration AppSource teste, valide et certifie votre application. 
6. Lorsque l’application est certifiée avec succès, examinez-la dans son environnement de test et mettez-la en production. 
7. L’application Power BI est répertoriée sur AppSource (elle « devient active »).
8. Dans Power BI, vous devez promouvoir le package de pré-production en production. Pour plus d’informations, consultez [Manage the template app release](https://docs.microsoft.com/power-bi/service-template-apps-create#manage-the-template-app-release) (Gérer la mise en production de l’application modèle).
